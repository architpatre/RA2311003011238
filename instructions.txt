This will be a coding assessment focusing on proficiency in REST APIs, Data Structures & Algorithms, and OOP concepts. Candidates are strictly prohibited from searching online or using language models like ChatGPT, DeepSeek, Claude, Github Copilot, etc.. Any evidence of plagiarism will lead to immediate disqualification. Note that existing online solutions may not accurately address the questions provided in this assessment.

Candidates will be presented with three different options based on their current skill level: Full Stack, Frontend, or Backend (choose one only). Those selecting Full Stack or Frontend must have React installed on their system. The Frontend option will also include a basic API server question. For Full Stack or Backend, any framework such as Net HTTP/Gin (Go), Express, etc., can be utilised. All candidates will have 3 hours to complete the assessment.

Ensure that their systems are preconfigured with all necessary tools installed, as no additional time will be provided for installation. If suitable IDE and API clients are not available, candidates must install them on their system. We recommend using VSCode and Postman/Insomnia. Use of Online IDEs or code editors is not allowed.

Candidates must push their code to GitHub and complete a Google form for test submission. When pushing code, everything should be included in one branch and one repository. Use the college roll number as the repository name, and use separate folders for each attempted question. Submissions will be disregarded if pushed to a different branch or repository. Additionally, candidates must add Node Modules, DS Store, etc., to the gitignore file before pushing the code.
